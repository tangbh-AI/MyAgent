MyAgent Web 端 — 架构设计总览

FastAPI + 静态前端 + 后台任务队列，复用全部现有核心模块

整体架构

系统数据流
浏览器 (HTML/JS)
│
├─ POST /chat { "message": "..." }
├─ GET /tasks/{id} (轮询状态)
└─ GET /download/{id}/report
FastAPI 应用 (myagent/web.py)
│
├─ 接收请求 → 创建任务 → 启后台线程
├─ 后台执行 5 阶段流水线
└─ 任务完成 → 标记状态 → 用户刷新
核心模块 (复用，不改动)
│
├─ ScriptGenerator → 参数提取+脚本生成
├─ AbaqusExecutor → subprocess 执行仿真
├─ ResultReader → ODB → results.json + PNG
└─ ReportGenerator → matplotlib → HTML报告

新增文件清单

文件 职责 行数预估

myagent/static/style.css 样式：左右分栏、消息气泡、任务卡片 ~200

任务状态机

📨 已提交 → 🔧 脚本生成中 → ⚙️ 仿真执行中 → 📊 提取结果中 → ✅ 完成
前端每 3 秒轮询 GET /tasks/{id}，状态变为"完成"时弹出刷新提醒
失败状态: ❌ 脚本生成失败 | ❌ 仿真执行失败 | ❌ 结果提取失败

API 端点设计

方法 路径 说明
GET / 前端页面
POST /api/chat 提交仿真描述 → 返回 task_id
GET /api/tasks/{task_id} 查询任务状态 {status, progress, result}
GET /api/tasks 列出所有任务（历史记录）
GET /api/models 列出可用模型
GET /download/{task_id}/report 下载 HTML 报告
GET /download/{task_id}/{filename} 下载云图 PNG / results.json 等

与现有 CLI 的关系

保持不变

  • myagent CLI 命令完全不受影响
  • config.yaml 配置共享
  • 所有核心模块 (generator/executor/result/report) 零改动

新增

  • myagent web 启动 Web 服务
